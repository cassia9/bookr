# PRD：預約管理模塊 - 新增/編輯與拖曳互動

**文檔版本**：1.0  
**建立日期**：2026-06-10  
**狀態**：已批准，執行中  
**所有者**：Vibe Coding 開發團隊

---

## 概述

### 問題陳述

目前的預約系統存在以下 UX 缺口：
- ❌ 無法快速新增預約（缺少統一的預約表單）
- ❌ 客戶選擇複雜（沒有既有客戶搜尋 + 新增客戶的合併流程）
- ❌ 預約編輯困難（需進入菜單，步驟繁瑣）
- ❌ 預約時間調整不便（無法直接拖曳改時間）
- ❌ 預約視圖單一（只有行事曆，缺少甘特圖維度）

### 目標

完成本功能後，用戶應能：
1. ✅ **快速新增預約** - 在 2 分鐘內完成新預約
2. ✅ **靈活調度** - 通過拖曳卡片快速改時間，無需進入編輯表單
3. ✅ **多維度查看** - 在行事曆和甘特圖間無縫切換
4. ✅ **高效管理** - 點擊卡片直接編輯，而非複雜菜單

## 核心功能

### 1️⃣ 新增預約流程

**入口**：
- 行事曆中點擊空白時段 → 打開新增預約表單
- 頂部「新增預約」按鈕 → 打開新增預約表單

**表單步驟**（Modal 分頁或單頁表單）：
```
Step 1: 選擇客戶
├─ 搜尋既有客戶（by 名字/電話）
└─ 新增客戶（快速輸入：名字、電話、備註）

Step 2: 選擇老師 & 課程
├─ 選擇老師（下拉或卡片）
└─ 選擇課程（該老師可提供的課程列表）

Step 3: 選擇預約時間
├─ 日期選擇
├─ 時間選擇（顯示老師當日的可用時段）
└─ 預約長度（根據課程自動帶入，可覆蓋）

Step 4: 確認 & 新增
└─ 顯示預約摘要，點擊「確認新增」
```

**驗證**：
- 客戶名字和電話必填
- 老師和課程必選
- 所選時段不能與既有預約衝突

---

### 2️⃣ 編輯預約流程

**入口**：
- 點擊行事曆/甘特圖上的預約卡片 → 打開編輯模式

**編輯模式**：
```
┌─────────────────────────────────────┐
│ 編輯預約                            │
├─────────────────────────────────────┤
│ 客戶：Ethan (可改)                  │
│ 老師：林老師 (可改)                 │
│ 課程：瑜伽課程 60 分鐘 (可改)      │
│ 時間：2026-06-15 14:00-15:00 (可改) │
│ 狀態：已確認 (可改)                 │
│ 備註：(可編輯)                      │
├─────────────────────────────────────┤
│ [取消] [儲存變更]                   │
└─────────────────────────────────────┘
```

**約束條件**：
- 時間改變時，實時檢查衝突
- 若改變老師，課程列表動態更新
- 不能編輯已完成的預約（只能查看）

---

### 3️⃣ 預約卡片拖曳

**功能**：
- 在行事曆或甘特圖中，點並拖曳預約卡片
- 拖曳時显示時間範圍提示（如「14:30 - 15:30」）
- 放開時自動更新預約時間到資料庫
- 若拖曳到有衝突的時段，顯示警告並不允許放開

**UX 細節**：
- Cursor 變為 `grab`，hover 時有視覺反饋
- 拖曳過程中，卡片半透明，背景變暗
- 實時驗證時段衝突，衝突區域變紅

---

### 4️⃣ 預約視圖

#### **行事曆視圖**（保留既有框架）

**三種模式**：
1. **月視圖**：格狀月曆，每格顯示當日預約總數（點擊進週視圖）
2. **週視圖**：時間軸（縱軸）× 日期（橫軸），顯示預約卡片
3. **日視圖**：單日詳細時段，精確到 30 分鐘粒度

**互動**：
- 支持按老師篩選
- 點擊空白時段 → 新增預約
- 點擊預約卡片 → 編輯或拖曳

---

#### **甘特圖視圖**（新增）

**結構**：
```
時間 ►  08:00  09:00  10:00  11:00  12:00  ...
老師 ▼
─────────────────────────────────────────────
林老師   [瑜伽]  [   伸展課   ]  [拳擊]
Kaku老師   [伸展課程]        [瑜伽]
sdf老師   
─────────────────────────────────────────────
```

**卡片內容**：
- 課程名字
- 客戶名字（小字）
- 預約時長（視覺寬度）
- 預約狀態（顏色區分）

**顏色方案**：
- 已確認 → 藍色
- 待確認 → 黃色
- 已完成 → 綠色
- 已取消 → 灰色

**互動**：
- 拖曳卡片改時間（同上）
- 點擊卡片編輯
- Hover 顯示完整信息（tooltip）
- 支持按老師篩選或搜尋

## 範圍邊界

### ✅ 在範圍內

- 新增預約（客戶選擇/新增、老師、課程、時間）
- 編輯預約（所有欄位可改，含時間）
- 刪除預約（軟刪除，前置確認）
- 拖曳改時間（行事曆和甘特圖）
- 行事曆三視圖（月/週/日）
- 甘特圖視圖（老師維度，30 分鐘粒度）
- 老師篩選
- 衝突檢測和提示

### ❌ 暫不在範圍內

- 線上支付（保留客戶到店付款）
- 客戶自動提醒通知（后续迭代）
- 預約評價系統（后续迭代）
- 批量操作（如批量改時間）
- 複製預約（如「複製上週的班表」）
- 同步到 Google Calendar / iCal
- 員工預排班功能

## 技術考慮

### 資料模型

**bookings 表**（擴展）：
```sql
CREATE TABLE bookings (
  id UUID PRIMARY KEY,
  store_id UUID NOT NULL,
  practitioner_id UUID NOT NULL,
  client_id UUID NOT NULL,
  service_id UUID NOT NULL,
  start_time TIMESTAMP NOT NULL,
  end_time TIMESTAMP NOT NULL,
  status TEXT DEFAULT 'pending',
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  deleted_at TIMESTAMP,
  FOREIGN KEY (store_id) REFERENCES stores(id),
  FOREIGN KEY (practitioner_id) REFERENCES practitioners(id),
  FOREIGN KEY (client_id) REFERENCES clients(id),
  FOREIGN KEY (service_id) REFERENCES services(id)
);

CREATE TABLE clients (
  id UUID PRIMARY KEY,
  store_id UUID NOT NULL,
  name TEXT NOT NULL,
  phone TEXT,
  email TEXT,
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  deleted_at TIMESTAMP,
  FOREIGN KEY (store_id) REFERENCES stores(id)
);
```

### RLS 政策

- 店家隔離：用戶只能看/編輯自己店家的預約
- 管理員可完全編輯，一般成員可自己的預約

### 拖曳實現

- 使用 **react-beautiful-dnd** 或原生 HTML5 Drag & Drop
- 拖曳時即時驗證 (debounce 防止過頻檢查)
- 衝突時 drop 不生效，視覺反饋

### 時段衝突檢測

```sql
-- SQL 查詢邏輯
SELECT COUNT(*) FROM bookings
WHERE practitioner_id = $1
  AND store_id = $2
  AND status IN ('pending', 'confirmed')
  AND start_time < $4    -- 新預約 end_time
  AND end_time > $3      -- 新預約 start_time
  AND id != $5           -- 排除自己（編輯時）
  AND deleted_at IS NULL
```

## 依賴與風險

| 項目 | 說明 | 影響 | 因應 |
|------|------|------|------|
| **clients 表** | 需要創建客戶資料表 | 高 | 第一階段遷移 |
| **拖曳庫選擇** | 選型（React Beautiful DnD vs 原生） | 中 | 評估 3 個方案 |
| **時段粒度** | 甘特圖最小單位（30 分鐘 vs 1 小時） | 低 | 根據業務需求決定 |
| **即時驗證性能** | 大量預約時檢查衝突可能變慢 | 中 | 添加資料庫索引 |
| **行事曆狀態同步** | 拖曳後行事曆需實時更新 | 中 | 使用 React 狀態管理 |
