# PRD：從業人員編輯表單 UX 優化

**文檔版本**：1.0  
**建立日期**：2026-06-09  
**最後更新**：2026-06-09  
**所有者**：Vibe Coding 開發團隊  
**狀態**：已批准，執行中

## 概述

### 問題陳述

當前從業人員（老師）編輯表單在以下方面存在 UX 瓶頸：
- 編輯模式與新增模式視覺上無區別，用戶不清楚當前操作上下文
- 課程選擇交互反饋不足，用戶難以一目瞭然已選課程
- 表單信息層級不清晰，優先級不明顯
- 按鈕狀態反饋不足，用戶不知道表單是否完整可提交

### 解決方案概要

通過以下改進優化表單 UX：
1. **上下文區分**：編輯模式顯示被編輯對象名字和更新時間
2. **課程反饋增強**：顯示已選課程計數（X/Y），選中卡片的視覺反饋
3. **信息層級優化**：更清晰的分組和視覺區分
4. **狀態控制**：動態啟用/禁用提交按鈕（至少需選 1 個課程）

---

## 設計方向

**美學風格**：精細極簡 + 功能優雅

特點：
- 清晰的信息層級分組
- 編輯 vs 新增的視覺差異化
- 課程選擇的流暢交互
- 微妙但有意義的動畫反饋

## 功能需求

### 1. 標題區域 - 編輯 vs 新增區分

#### 新增模式
```
┌──────────────────────────────────────┐
│ 新增老師                             │
│ 新增一位老師到系統                   │
└──────────────────────────────────────┘
```

#### 編輯模式（視覺區分）
```
┌──────────────────────────────────────┐
│ 編輯老師                             │
│ 更新 林老師 的基本資料和課程指派     │
│ 最後更新：2026-06-09 14:30           │
└──────────────────────────────────────┘
```

**實現細節**：
- 新增模式：使用 ➕ 圖標
- 編輯模式：使用 ✏️ 圖標 + 被編輯人名字 + 最後更新時間
- 字體大小：標題 28px，副標題 14px

---

### 2. 基本資訊區塊

**包含欄位**：
- 老師名字（必填）
- 識別顏色（必填）
- 簡介（選填）

#### 2.1 名字輸入框
```
標籤：老師名字 *
輸入框：[         ]
提示文本：例：林老師
```

#### 2.2 識別顏色選擇（改進版）
```
標籤：識別顏色 *

顏色選擇區：
[紫] [藍] [綠] [紅] [橙]
[粉] [青] [靛] ✓

展示：已選：紫色
當前名字在系統中會顯示為紫色
```

**改進說明**：
- ❌ 移除色票符號（◆），只顯示純色塊
- ✅ 選中時顯示 ✓ 圖標
- ✅ 顯示已選顏色的文字說明

---

### 3. 課程選擇區塊（核心改進）

#### 3.1 標題與計數器
```
標籤：可預約課程 *  
計數器：2/8 已選  ← 動態更新
```

#### 3.2 課程列表

**未選狀態**：
```
┌─────────────────────────────────────┐
│ ☐ 瑜伽課程         60 分鐘  ¥800    │
│   邊框：淺灰 (border-slate-200)     │
│   背景：白色                         │
│   文字：slate-900                    │
└─────────────────────────────────────┘
```

**選中狀態**：
```
┌─────────────────────────────────────┐
│ ☑ 伸展課程 ✓       45 分鐘  ¥600    │
│   邊框：黑色加粗 (border-black)     │
│   背景：淺黑 (bg-black/5)           │
│   文字：slate-900 (加粗)             │
└─────────────────────────────────────┘
```

**交互反饋**：
- Hover 時：陰影加深 + 背景色變淡
- 點擊時：立即視覺反饋（顏色+邊框變化）
- 微動畫：課程卡片 hover 時微妙縮放 (1.02x)

**功能說明**：
- 支持多選（至少 1 個，無上限）
- 已選課程計數器即時更新
- 清晰的選中/未選視覺區分

---

### 4. 簡介區塊

```
標籤：簡介（選填）

文本區：
[                                    ]
[  輸入老師的簡短介紹或專長...      ]
[                                    ]

提示文本：例：擅長運動傷害、肌筋膜放鬆...
```

---

### 5. 按鈕組（狀態控制）

#### 5.1 按鈕狀態

**必填項未完成時**（禁用狀態）：
```
┌────────────┬────────────────┐
│ 取消 (灰)  │ 新增老師 (禁用灰) │
└────────────┴────────────────┘
提示：需選擇至少 1 個課程
```

**必填項完成時**（啟用狀態）：
```
┌────────────┬────────────────┐
│ 取消       │ 新增老師 (黑)    │
└────────────┴────────────────┘
Hover：按鈕有微妙放大效果
```

**提交中**（加載狀態）：
```
┌────────────┬─────────────────┐
│ 取消 (禁用) │ ⏳ 新增中... (禁用) │
└────────────┴─────────────────┘
```

#### 5.2 按鈕交互規則

| 狀態 | 條件 | 樣式 | 可點擊 |
|-----|------|------|--------|
| 禁用 | 未選課程 OR 名字為空 | 灰色 (bg-slate-300) | ❌ |
| 啟用 | 已選 ≥1 課程 AND 名字非空 | 黑色 (bg-black) | ✅ |
| 加載 | 正在提交 | 灰色 + 加載動畫 | ❌ |
| 成功 | 提交成功後短暫顯示 | 綠色 + ✓ 圖標 | ❌ |

## 視覺設計細節

### 色彩體系
- **主色**：黑色 (#000000) - 主要交互
- **次色**：綠色 (#22C55E) - 成功狀態
- **中立色**：灰色系 (slate-*) - 背景和禁用
- **邊框**：slate-200（未選）/ black（已選）

### 間距與排版
- 卡片間距：12px (space-y-3)
- 內邊距：16px (px-4 py-3)
- 標題字號：16px 加粗
- 副文本：14px 灰色

### 動畫與過渡
- 按鈕 Hover：縮放 1.02x，過渡時間 150ms
- 課程卡片 Hover：縮放 1.02x，陰影加深
- 狀態變化：150-200ms 平滑過渡
- 禁用→啟用：漸進式色彩變化
