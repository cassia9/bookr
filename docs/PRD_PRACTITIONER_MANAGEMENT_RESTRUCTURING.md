# PRD：從業人員管理功能重新組織

**版本**: 1.0  
**日期**: 2026-06-05  
**狀態**: 建議中  
**優先級**: 高 (影響 IA 和導航)

---

## 📋 問題陳述

目前，**從業人員管理功能嵌入在預約管理頁面的側邊欄中**，這導致：

### 現有問題
1. **信息架構不清晰** - 用戶不知道從業人員管理是獨立功能還是預約工具
2. **導航層級錯誤** - 從業人員是核心資源，應與預約同級別
3. **視覺層級混亂** - 側邊欄佔用預約管理的工作區
4. **擴展性限制** - 難以為從業人員添加獨立功能（績效、工資、班表等）
5. **用戶流不直觀** - 需要進入預約管理才能管理從業人員

### 正確的心智模型
```
預約系統
├─ 預約管理        ← 查看和管理預約
├─ 從業人員管理    ← 獨立的資源管理（新增/編輯/刪除/休假）
├─ 課程管理        ← 管理課程
├─ 客戶管理        ← 管理客戶
└─ 報表分析        ← 數據分析
```

---

## 🎯 目標和成功指標

### 目標
- 將從業人員管理升級為**獨立的頂級導航項目**
- 提供完整的從業人員生命週期管理界面
- 簡化預約管理頁面（專注於預約日程）
- 改進用戶心智模型和信息架構

### 成功指標
| 指標 | 基準 | 目標 |
|------|------|------|
| **導航清晰度** | 側邊欄佔用空間 | 專用頁面 100% 寬度 |
| **頁面加載時間** | ~2.5s | <2s (減少渲染負擔) |
| **用戶操作步驟** | 3 步 (進入預約→點擊側邊欄→操作) | 1 步 (直接進入) |
| **可擴展性** | 6 個功能點 | 12+ 功能點 (預留空間) |

---

## 💡 解決方案概述

### 架構變更

#### 現狀
```
/admin/booking
└─ BookingManagement
   ├─ PractitionerList (側邊欄)
   │  ├─ PractitionerForm (Modal)
   │  └─ PractitionerLeaveManager (Modal)
   ├─ CalendarPage
   └─ GanttPage
```

#### 優化後
```
/admin/practitioners (NEW)
└─ PractitionerManagement (NEW)
   ├─ PractitionerList (主表格)
   ├─ PractitionerForm (Modal / Drawer)
   └─ PractitionerLeaveManager (Modal)

/admin/booking (已優化)
└─ BookingManagement (簡化)
   ├─ 可選：快速篩選 (按從業人員)
   ├─ CalendarPage
   └─ GanttPage
```

### 關鍵變更

#### 1. 新頁面：PractitionerManagement
```typescript
// 在 /admin/practitioners 路由
// 提供完整的從業人員管理界面
// - 表格視圖（名字、顏色、課程、狀態）
// - 批量操作（多選）
// - 快速統計（總數、活躍、停用）
// - 搜尋和篩選
```

#### 2. 簡化的 BookingManagement
```typescript
// 保留可選的從業人員快速篩選
// 但移除完整的側邊欄管理面板
// 新增：快速導航到從業人員管理
```

#### 3. 主導航菜單更新
```
新增菜單項：
- 從業人員管理 (icon: Users / UserCheck)
  └─ 進入 /admin/practitioners
```

## 📐 功能需求

### FR-1: PractitionerManagement 頁面

**頁面結構：**
```
┌─────────────────────────────────────────────┐
│ 頂部欄                                       │
│ 標題: 從業人員管理                          │
│ 操作: 新增老師, 篩選, 匯出                  │
├─────────────────────────────────────────────┤
│ 統計卡片 (4 列)                             │
│ ├─ 總數: 12                                 │
│ ├─ 活躍: 10                                 │
│ ├─ 停用: 2                                  │
│ └─ 今日休假: 1                              │
├─────────────────────────────────────────────┤
│ 搜尋和篩選                                   │
│ ├─ 搜尋框: 名字/課程                        │
│ └─ 篩選: 狀態, 課程                         │
├─────────────────────────────────────────────┤
│ 從業人員表格                                 │
│ ├─ 名字 | 顏色 | 課程 | 預約數 | 狀態 | 操作
│ ├─ 操作: 編輯, 刪除, 管理休假, 查看預約    │
│ └─ 支持排序和分頁                           │
└─────────────────────────────────────────────┘
```

**需求清單：**
1. ✅ 顯示所有從業人員（分頁或虛擬化）
2. ✅ KPI 卡片（總數、活躍、停用、今日休假）
3. ✅ 搜尋和篩選（名字、課程、狀態）
4. ✅ 表格操作（編輯、刪除、管理休假）
5. ✅ 快速查看預約（導航到該從業人員的預約）
6. ✅ 批量操作（停用、啟用、刪除）※ 未來
7. ✅ 匯出為 CSV/PDF ※ 未來

### FR-2: 導航菜單更新

**左側邊欄菜單：**
```
主菜單
├─ 儀表板
├─ 預約管理       ← 簡化版
├─ 從業人員管理   ← 新增 (此處)
├─ 課程管理
├─ 客戶管理
└─ 報表分析
```

**實現細節：**
- 新增菜單項，圖標使用 Lucide `Users` 或 `UserCheck`
- 底色深，hover 狀態清晰
- 當前頁面高亮

### FR-3: BookingManagement 簡化

**移除的功能：**
- ❌ 左側邊欄的完整從業人員列表
- ❌ 側邊欄的新增/編輯/刪除操作

**保留/新增：**
- ✅ 頂部的從業人員篩選快速按鈕（6 個常用老師）
- ✅ 快速導航按鈕：「管理從業人員」→ `/admin/practitioners`
- ✅ 專注於日程和預約

## 🔄 用戶流變更

### 現狀流程
```
進入預約管理
  ↓
點擊側邊欄老師
  ↓
展開操作按鈕
  ↓
點擊「編輯」/「刪除」/「管理休假」
  ↓
Modal 彈出
```

### 優化後流程
```
方式 A：直接進入從業人員管理
進入主菜單
  ↓
點擊「從業人員管理」
  ↓
進入 PractitionerManagement 頁面
  ↓
在表格中找到老師
  ↓
點擊行操作按鈕
  ↓
Modal 彈出

方式 B：從預約管理快速操作
進入預約管理
  ↓
點擊頂部「管理從業人員」按鈕
  ↓
跳轉到 PractitionerManagement 頁面
```
