# PRD：統一預約管理系統

**版本**: 1.0  
**狀態**: Ready for Implementation  
**優先級**: High (P1)  
**更新日期**: 2026-06-04  

---

## 1️⃣ 問題陳述

### 現狀

目前系統將「行事曆」和「甘特圖」分開為兩個獨立頁面：
- 行事曆頁面：月/週/日視圖，展示預約卡片
- 甘特圖頁面：按從業人員分組的時間表

### 問題

1. **導航混亂**：用戶需要在兩個不同的菜單項之間切換
2. **上下文切換成本高**：查看同一批預約需要切換頁面
3. **從業人員管理缺失**：無法快速新增或管理被預約的老師
4. **從業人員狀況不可見**：在主視圖中看不到哪些老師被預約了
5. **組織混亂**：相關功能分散在不同位置

### 為什麼重要

- **提高效率**：統一界面讓用戶一次查看所有預約信息
- **降低學習曲線**：新用戶更容易理解功能組織
- **支持業務流程**：可以同時管理預約和從業人員
- **提升用戶體驗**：減少頁面切換和上下文損失

---

## 2️⃣ 目標與成功指標

### 目標

建立一個統一的「預約管理」中心，集成行事曆、甘特圖和從業人員管理功能，讓經營者可以在一個頁面中全面掌控預約狀況。

### 成功指標

| 指標 | 基線 | 目標 | 衡量方法 |
|------|------|------|---------|
| 頁面切換次數 | 2 次 | 0 次 | 用戶研究/測試觀察 |
| 視圖切換時間 | <2 秒 | <500ms | 性能測試 |
| 從業人員新增時間 | 無此功能 | <30 秒 | 用戶測試 |
| UI 一致性得分 | N/A | 100% | 設計審查 |
| 功能覆蓋率 | 2 個視圖 | 3 個視圖 + 管理 | 功能清單檢查 |

## 3️⃣ 解決方案概述

### 高層架構

```
預約管理（統一頁面）
├── 左側面板
│   ├── 視圖切換（Calendar / Gantt）
│   ├── 日期導航
│   └── 從業人員管理
│       ├── 從業人員列表
│       ├── 新增老師 Button
│       └── 老師被預約狀況指示
│
└── 主視圖區域
    ├── 行事曆模式
    │   ├── 月視圖
    │   ├── 週視圖
    │   └── 日視圖
    │
    └── 甘特圖模式
        └── 按老師的時間表
```

### 核心功能

1. **統一導航** - 合併行事曆和甘特圖為單一菜單項
2. **視圖切換** - 在同一頁面快速切換行事曆/甘特圖
3. **從業人員面板** - 側邊欄展示所有從業人員及其預約狀況
4. **新增從業人員** - 快速新增被預約的老師（不需要進入設定頁）
5. **視覺指示** - 老師被預約情況的實時更新

## 5️⃣ 設計規範（Design Guideline 遵守）

### 顏色系統

參考 AdminLayout 設計：

```
主色：Indigo (bg-indigo-600, text-indigo-700)
次色：Slate (text-slate-600, bg-slate-50)
背景：white, bg-gray-50
邊框：border-slate-200

狀態顏色：
- 成功：green-100 / green-900
- 警告：yellow-100 / yellow-900
- 完成：blue-100 / blue-900
- 錯誤：red-100 / red-900
- 中立：gray-100 / gray-900
```

### 間距和排版

```
側邊欄寬度：240px
邊距：p-4, p-6
間隔：gap-3, gap-4
頭級標題：text-3xl font-bold
副標題：text-lg font-semibold
正文：text-sm text-slate-600
```

### 圖標

- 使用 `lucide-react` 圖標庫
- 大小：16-24px（根據上下文）
- 顏色：繼承文本顏色

### 交互狀態

```
按鈕：
- 正常：bg-white, border, text-slate-600
- 懸停：bg-slate-50, text-slate-900
- 活躍：bg-indigo-50, text-indigo-700
- 禁用：opacity-50, cursor-not-allowed

輸入框：
- 邊框：border-slate-300
- 焦點：border-indigo-600, ring-2, ring-indigo-100
- 錯誤：border-red-500
```

### 響應式設計

```
Desktop：側邊欄 + 主視圖 (並行)
Tablet：側邊欄可折疊 (< 1024px)
Mobile：全屏垂直堆疊 (< 640px)
```

### 前端架構

```
頁面：/src/pages/admin/BookingManagement.tsx（新統一頁面）

替換：
- /admin/calendar → /admin/bookings (重定向)
- /admin/gantt → /admin/bookings (重定向)

子組件：
- BookingManagementLayout.tsx (主容器)
- PractitionerPanel.tsx (側邊欄)
- BookingCalendarView.tsx (行事曆視圖)
- BookingGanttView.tsx (甘特圖視圖)
- BookingDetailDrawer.tsx (詳情抽屜)
- PractitionerModal.tsx (新增老師 Modal)
```

### 數據流

```
1. 載入從業人員列表 (practitioners)
2. 根據選中日期和視圖模式載入預約
3. 計算從業人員預約狀況 (count, next_available)
4. 實時更新（Supabase Realtime 訂閱）

API：
- GET /bookings?practitioner_id=&date_range=
- GET /practitioners
- POST /practitioners (新增)
- PUT /bookings/:id (編輯)
```

### 性能優化

```
- 虛擬化長列表（if > 50 老師）
- 按需加載預約（日期/視圖切換時）
- 快取從業人員列表（60 秒）
- 模式切換動畫使用 CSS transition（無重排）
```

### 相容性

- 現代瀏覽器（Chrome, Firefox, Safari, Edge）
- 移動設備（iOS Safari, Chrome Mobile）
- 無 IE 支持要求
